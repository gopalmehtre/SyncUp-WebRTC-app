# SyncUP - Real-time Video Conferencing App

SyncUP is a full-stack web application that provides real-time video and audio communication, screen sharing, and chat functionalities, built with the MERN stack (MongoDB, Express, React, Node.js) and WebRTC.


![Landing Page](/frontend/public/mobile.png)

## Features

-   **User Authentication**: Secure registration and login system for users.
-   **Guest Access**: Join meetings instantly as a guest without registration.
-   **Create & Join Meetings**: Start a new meeting or join an existing one using a unique meeting code.
-   **Real-time Communication**: High-quality video and audio streaming using WebRTC.
-   **Screen Sharing**: Share your screen with other participants in the meeting.
-   **In-Meeting Chat**: Real-time text messaging with all participants during a call.
-   **Meeting History**: Registered users can view their past meeting activities.
-   **Dynamic Video Layout**: The video grid adjusts dynamically based on the number of participants.

## Getting Started

Follow these instructions to get a copy of the project up and running on your local machine for development and testing purposes.

### Prerequisites

-   Node.js (v14 or later)
-   npm
-   MongoDB (local instance or a cloud-based service like MongoDB Atlas)

### Backend Setup

1.  **Navigate to the backend directory:**
    ```sh
    cd backend
    ```

2.  **Install dependencies:**
    ```sh
    npm install
    ```

3.  **Create an environment file:**
    Create a `.env` file in the `backend` directory and add the following variables:
    ```
    // filepath: backend/.env
    MONGODB_URI=<YOUR_MONGODB_CONNECTION_STRING>
    PORT=8000
    ```

4.  **Run the server:**
    ```sh
    npm run dev
    ```
    The backend server will start on `http://localhost:8000`.

### Frontend Setup

1.  **Navigate to the frontend directory:**
    ```sh
    cd frontend
    ```

2.  **Install dependencies:**
    ```sh
    npm install
    ```

3.  **Update the server endpoint (for local development):**
    In `frontend/enviroment.js`, set `IS_PROD` to `false`.
    ````javascript
    // filepath: frontend/enviroment.js
    let IS_PROD = false; // Set to false for local development
    const server = IS_PROD ?
        "https://syncup-webrtc-app.onrender.com" :
        "http://localhost:8000"


    export default server;
    ````

4.  **Run the client:**
    ```sh
    npm run dev
    ```
    The React development server will start, and the application will be accessible at `http://localhost:3000`.

## API Endpoints

The backend exposes the following REST API endpoints under the `/api/v1/users` route:

-   `POST /register`: Register a new user.
-   `POST /login`: Log in an existing user.
-   `POST /add_to_activity`: Add a meeting to a user's history.
-   `GET /get_all_activity`: Retrieve the meeting history for a logged-in user.

## Socket.IO Events

Real-time communication is handled via Socket.IO with the following events:

-   `join-call`: A user joins a specific meeting room.
-   `user-joined`: Notifies clients in a room that a new user has joined.
-   `user-left`: Notifies clients that a user has disconnected.
-   `signal`: Relays WebRTC signaling messages (offers, answers, ICE candidates) between peers.
-   `chat-message`: Broadcasts a chat message to all users in the room.
